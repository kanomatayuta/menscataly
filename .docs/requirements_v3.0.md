## 3. SEO戦略 (v3.0 新規)

### 3.1 Google AI Overview 対策

リサーチで判明した重要事項：
- **医療系クエリのAI Overview**: 治療・手順系は100%表示 (2023年の45%から55pt増)
- **ローカル/プロバイダー系**: AI Overview 0% (完全排除)
- **AI Overview表示時のCTR低下**: 情報系クエリで約61%低下
- **AI Overview引用サイトのCTR**: 逆に約35%向上

**戦略**:
1. AI Overviewに**引用される側**を目指す → 構造化データ、明確な回答、E-E-A-T強化
2. ローカルKW (「[地域名] AGAクリニック」) は従来型SEOで勝負
3. 比較表・FAQ構造化で引用確率を最大化

### 3.2 AI生成コンテンツのSEO対策

Googleの最新ガイドライン:
- AI生成自体はペナルティ対象外
- **ただし**: 大量生成で価値を付加しない場合はスパムポリシー違反
- 最新品質評価ガイドラインで「AI生成に最低評価の可能性」を明確化
- **「人間による有用な補正」**が必須

**対応策**:
1. Phase 0-1: 全記事を人間が確認後に公開
2. Phase 2以降: 品質スコア80点以上のみ自動公開、それ以下は人間レビュー
3. 各記事に独自の付加価値 (体験談トーン、独自比較データ、監修者コメント) を自動挿入
4. 重複率チェック・オリジナリティスコアを品質基準に組み込む

### 3.3 E-E-A-T 強化策 (YMYL対応)

AI Overview時代ではE-E-A-Tの重要度が**10倍**に増幅:

| 要素 | 実装方法 |
|------|---------|
| Experience | 「実際に使ってみた」トーンの記事構成、Before/After記述 |
| Expertise | 医師監修表記、資格・専門分野の明示 |
| Authoritativeness | 厚労省・学術論文引用、構造化データ (MedicalWebPage) |
| Trustworthiness | 運営者情報ページ、最終更新日自動更新、免責事項テンプレート |

## 4. コンプライアンス (v3.0 強化)

### 4.1 薬機法対応 (v2.0踏襲 + 強化)
- 薬機法第66条 (誇大広告禁止) ・第67条 (承認前広告禁止) 準拠
- **対象: 「何人も」** → アフィリエイターも直接の規制対象
- NG表現自動検出・OK表現置換システムの精度向上

### 4.2 景表法・ステマ規制 (v3.0 新規)
- **2023年10月施行**: ステマが景品表示法違反に
- **2024年12月改定**: 健康食品関連の虚偽・誇大表示の具体例拡充、アフィリエイト広告規制強化
- **対応**: 全記事にPR表記・広告表記を自動挿入、アフィリエイトリンクにrel="sponsored"属性付与

### 4.3 Cookie・トラッキング規制対応 (v3.0 新規)

| ブラウザ | サードパーティCookie | ファーストパーティCookie |
|---------|--------------------|-----------------------|
| Safari | 完全ブロック | 最大7日で削除 (ITP) |
| Firefox | 完全ブロック | 制限あり |
| Chrome | ユーザー選択制 (2025年4月方針変更) | 制限なし |

**対応策**:
- 各ASPのITP対応タグを確実に実装
- サーバーサイドトラッキングの検討
- ファーストパーティデータ (GA4 + Supabase) を収益分析の主軸に

### 8.2 エージェント詳細

#### Lead Agent (指揮官)
- **役割**: プロジェクト全体の統括、タスク分解、各エージェントへの指示、成果物レビュー
- **ツール**: Claude Code メインセッション
- **モデル**: Claude Opus 4.6
- **ワークフロー**: タスク → 分解 → 各エージェントに並列委任 → 結果統合

#### Frontend Agent
- **役割**: Next.js 16アプリケーション開発、UI/UXデザイン、コンポーネント設計
- **担当領域**:
  - Next.js 16 (Turbopack, PPR) セットアップ
  - 記事テンプレート・コンポーネント設計
  - SEO最適化 (構造化データ、OGP動的生成)
  - レスポンシブデザイン
  - Core Web Vitals最適化
- **ワークツリー**: `feature/frontend-*`

#### Backend Agent
- **役割**: AIパイプライン構築、API連携、データベース設計
- **担当領域**:
  - Cloud Scheduler / Cloud Run パイプライン
  - Claude API連携 (記事生成・分析)
  - microCMS API連携
  - Supabase スキーマ設計
  - BigQuery / Looker Studio連携
  - GA4 / GSC Data API連携
- **ワークツリー**: `feature/backend-*`

#### Content Agent
- **役割**: 記事生成エンジン、SEO最適化、コンプライアンスチェック
- **担当領域**:
  - 記事生成プロンプト設計
  - 薬機法チェッカー実装
  - ステマ規制対応 (PR表記自動挿入)
  - ASPリンク管理・自動挿入
  - サムネイル生成 (Ideogram連携)
  - 内部リンク自動設計
  - E-E-A-T要素自動挿入
- **ワークツリー**: `feature/content-*`

#### QA & Review Agent
- **役割**: テスト、品質保証、コードレビュー
- **担当領域**:
  - ユニットテスト / E2Eテスト
  - パフォーマンステスト (Lighthouse)
  - セキュリティレビュー
  - アクセシビリティチェック
  - 記事品質スコアリングの検証
  - SEO監査
- **ワークツリー**: `feature/qa-*`
